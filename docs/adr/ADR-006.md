# ADR-006: Async Test Runner Pattern

## Status
Accepted

## Context
Test suites on large repos take 10-20 minutes. MCP tools cannot block
for that duration. A fire-and-forget pattern with log retrieval is needed.

## Decision
Two MCP tools: `run_tests` and `get_test_log`.
`run_tests` spawns a detached subprocess using `shell=True`, redirects
stdout+stderr to `docs/state/openagent_test.log`, writes PID to
`docs/state/openagent_test.pid`, and returns within one second.
`get_test_log` reads the log file and checks PID liveness via tasklist
(Windows) with ps fallback (Unix).

Test command resolution order:
  1. `test_cmd:` line in AGENT_CONTRACT.md (wins unconditionally)
  2. File-based detection (Cargo.toml, pyproject.toml, package.json, go.mod)
  3. None — error returned, never guessed

## Consequences
- `shell=True` is permanent for test command execution (ADR-006 locked)
- Log path `docs/state/openagent_test.log` is the canonical output location
- Max tail is 200 lines — callers cannot exceed this
- `run_tests` never blocks — any error surfaces in the return dict
- Auto-triggering tests during `assess_repo` is permanently out of scope
